# Build Setup Guide

## Overview

The conference mode application now uses **webpack** to bundle frontend dependencies locally instead of relying on CDNs:

- **LemonadeJS** - Reactive UI framework
- **Prism.js** - Syntax highlighting for Java code
- **Shared CSS** - Unified styling across player and presenter views

## Build Process

### Installation

```bash
npm install
```

This installs:
- **Dependencies**: `express`, `ws`, `sqlite3`, `uuid`, `lemonadejs`, `prismjs`
- **DevDependencies**: `webpack`, `webpack-cli`, `style-loader`, `css-loader`

### Building

```bash
npm run build
```

Generates bundled files:
- `public/dist/player.bundle.js` - Player view with all dependencies
- `public/dist/presenter.bundle.js` - Presenter view with all dependencies

Webpack will:
1. Import and bundle LemonadeJS and Prism.js
2. Include Prism CSS themes
3. Bundle shared CSS from `src/shared.css`
4. Minify and optimize for production

### Generating Quiz JSON Assets

The presenter/player views load quiz data from `code.json`, `descriptions.json`, and optionally `object-sizes.json`.
Generate them with:

```bash
python3 game/generate_quiz.py --game both --base-url http://localhost:3000/
```

This will generate JSON in `game/dist/` and automatically copy the JSON files into:

```
game/conference/public/
```

so the conference server can serve them at `/code.json`, `/descriptions.json`, and `/object-sizes.json`.

### Watch Mode (Development)

```bash
npm run build:watch
```

Automatically rebuilds bundles when source files change.

### Running the Server

```bash
npm start
```

Runs `npm run build` then starts the server on port 3000.

## How It Works

### HTML Files (Minimal)

Both `player.html` and `presenter.html` are now minimal:
- No inline styles or scripts
- No CDN links
- Just load the corresponding bundle:
  ```html
  <div id="app"></div>
  <script src="dist/player.bundle.js"></script>
  ```

### Entry Points (`src/player.js` and `src/presenter.js`)

Each entry point:
1. Imports LemonadeJS, Prism, and CSS
2. Contains all game logic
3. Gets bundled by webpack into a single file

### Shared CSS (`src/shared.css`)

Contains all styling that's imported by both bundles:
- Layout & grid systems
- Typography
- Colors & themes
- Component styles
- Responsive design rules

### Webpack Configuration

`webpack.config.js` defines two build targets:

1. **Player bundle**: `src/player.js` в†’ `public/dist/player.bundle.js`
2. **Presenter bundle**: `src/presenter.js` в†’ `public/dist/presenter.bundle.js`

Both use:
- `style-loader` to inject CSS into bundles
- `css-loader` to process CSS imports
- Production mode for optimization

## Development Workflow

1. **Edit source files** in `src/` (JavaScript and CSS)
2. **Run build** with `npm run build` or use watch mode
3. **Start server** with `npm start`
4. **Refresh browser** to see changes

For development with live reload:
```bash
npm run build:watch &
npm start
```

## Production Deployment

1. Run `npm install --production` to skip dev dependencies
2. Run `npm run build` to generate optimized bundles
3. Deploy the entire `game/conference/` directory
4. Set `ADMIN_SECRET` environment variable
5. Run `npm start`

## Troubleshooting

### Bundles not updating
Make sure webpack is running:
```bash
npm run build
```

### Prism syntax highlighting not working
Ensure `prismjs/themes/prism-tomorrow.css` is imported in the entry point and bundled with style-loader.

### Styles not applying
Check that `src/shared.css` is imported and style-loader is configured in webpack.

### Module not found errors
Verify all imports in `src/player.js` and `src/presenter.js` match the installed packages.
