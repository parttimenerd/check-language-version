### Summary


Adds `record` classes for concise immutable data carriers with generated accessors and `equals`/`hashCode`/`toString`.

### Details

A `record` is a special kind of class intended to model **data**. Its primary state is declared in the *record header* (the component list). From that header, the compiler generates:

- a `private final` field for each component
- a public accessor method with the same name as the component
- implementations of [`Object.equals(Object)`](https://docs.oracle.com/en/java/javase/21/docs/api/java.base/java/lang/Object.html#equals(java.lang.Object)), [`Object.hashCode()`](https://docs.oracle.com/en/java/javase/21/docs/api/java.base/java/lang/Object.html#hashCode()), and [`Object.toString()`](https://docs.oracle.com/en/java/javase/21/docs/api/java.base/java/lang/Object.html#toString())

Records are implicitly `final` and extend [`java.lang.Record`](https://docs.oracle.com/en/java/javase/21/docs/api/java.base/java/lang/Record.html). You can add:

- extra fields (typically `static`)
- methods
- constructors:
  - a *canonical constructor* matching the record components
  - a *compact constructor* `public Name { ... }` to validate/normalize components

Records work well with pattern matching features and improve readability when a class is primarily “just data”.

### Example

```java
// Define a record and validate its components
record Point(int x, int y) {
    public Point {
        if (x < 0 || y < 0) {
            throw new IllegalArgumentException("x/y must be non-negative"); // validation
        }
    }
}

class Example {
    void demo() {
        Point p = new Point(1, 2);
        int x = p.x(); // accessor generated by the compiler
    }
}
```

### Historical

Standardized in Java 16 (after being previewed in earlier releases) as a language feature to reduce boilerplate for simple immutable data classes.
